# 🏃 Prototype (프로토타입) - 개발 완료 보고서 (시연 및 PPT/보고서용)

본 문서는 해커톤 시연, 프로젝트 보고서 및 PPT 작성을 위해 **Prototype** 애플리케이션의 핵심 개념, 기술 스택, 주요 기능, 데이터베이스 구조 및 시연 시나리오를 요약한 개발 완료 요약서입니다.

---

## 1. 프로젝트 개요 (Concept)

* **앱 이름**: `Prototype`
* **주요 타겟**: 운동 의지가 부족한 현대 학생 및 직장인 (MZ세대)
* **핵심 슬로건**: *"낙오하면 박제된다!"* 킹받는 AI 경고 문자와 웹소설 보상 시스템을 결합한 연대책임 피트니스 서비스.
* **차별화된 강점**:
  1. **손실 회피 동기부여**: 단순히 목표 달성 시 리워드를 주는 것을 넘어, **실패 시 지인에게 킹받는 AI 경고 문자(SMS)가 발송**되는 벌칙형 페널티 설계.
  2. **연대책임 (듀오 모드)**: 친구와 함께 연동하여 공동 운명체가 됩니다. 둘 중 한 명이라도 목표를 채우지 못하면 경고 문자 발송 가능성이 활성화되어 피어 프레셔(Peer Pressure)를 극대화합니다.
  3. **게이미피케이션 & 웹소설 보상**: 걸음 수에 비례하여 트렌디한 AI 생성 웹소설(아이돌 로맨스, 드라마 빙의물 등) 챕터가 해금되며, 모은 포인트로 히든 선택지를 열어 스토리에 영향을 줍니다.

## 2. 핵심 기능 상세

### 🔑 A. 사용자 인증 & 개인화
* **Firebase Auth 기반 인증**: 이메일/비밀번호 간편 회원가입 및 로그인 지원.
* **임의 닉네임 자동 생성**: 신규 회원가입 시 `날렵한토끼581`, `끈기있는펭귄302` 와 같은 귀여운 조합의 랜덤 닉네임이 자동으로 선점 부여됩니다.
* **프로필 설정**: 설정 탭에서 언제든지 닉네임 및 경고 문자 발송 대상 연락처를 수정·관리할 수 있습니다.

### 🚶 B. 실시간 걸음 수 측정 & 치팅 방지 (Anti-Cheat)
* **Pedometer 연동**: 모바일 기기의 네이티브 센서를 이용해 실제 걸음 수를 백그라운드에서 정확히 추적합니다.
* **꼼수 방지 엔진 (자이로 센서)**: 디바이스의 가속도 센서를 분석하여 손으로 기기를 억지로 흔드는 꼼수를 감지합니다. 감지 시 화면에 붉은색 돼지 경고 팝업이 뜨며 **5초 동안 걸음 수 적립이 즉시 동결**됩니다.

### 👥 C. 2인 연대책임 듀오 챌린지
* **공동 운명체 매칭**: 상대방의 이름과 연락처를 입력해 듀오 챌린지를 활성화합니다.
* **실시간 상호 모니터링**: 홈 화면에 나와 파트너의 오늘 걸음 수가 실시간으로 비교 표시되어 서로에게 러닝 페이스메이커 역할을 수행합니다.

### 🚨 D. 킹받는 AI 경고 문자 발송
* **AI 작문 엔진**: 누적 걸음 수, 달성률, 사용자 닉네임을 파악해 OpenAI/LLM 모델을 통해 대상 수신자가 읽었을 때 극도로 킹받고 미안하게 만드는 **맞춤형 사죄/고발 문자**를 실시간으로 작문해 발송합니다.
* **실드 방어**: 포인트 상점에서 구매한 실드(`🛡️`)가 활성화되어 있다면, 목표를 달성하지 못해도 경고 발송을 1회 방어할 수 있습니다.

### 📖 E. AI 웹소설 보상 시스템 (스토리 모드)
* **인기 서사 테마 제공**: 요즘 학생들이 열광하는 웹소설 테마(드라마 속 빙의물, 아이돌 고백물 등)가 무작위로 선택되어 시작됩니다.
* **걸음 비례 해금**: 누적 `1,000보`가 쌓일 때마다 다음 챕터의 해금 선택지가 열려 스토리가 이어집니다.
* **선택지 분기**: 사용자의 행동 선택에 따라 이야기가 멀티 엔딩으로 전개되며, 포인트로 해금하는 **🔑 히든 선택지**를 통해 운명을 개척할 수 있습니다.

### 🛒 F. Toss 스타일 포인트 숍 (상점)
* **포인트 적립**: 일일 목표 달성(+100P) 및 걸음 마일스톤 돌파(+50P) 시 포인트를 적립합니다.
* **스토어 아이템**:
  1. `🛡️ 하루 벌칙 방지 실드` (100 P): 경고 발송 면제권.
  2. `🔑 히든 스토리 열쇠` (150 P): 스토리 탭에서 잠긴 특수 분기(히든 스토리)를 해제하는 열쇠.
  3. `👟 행운의 랜덤 박스` (50 P): **하루에 단 한 번만 열 수 있는 랜덤 박스**로, 10P ~ 200P 사이의 포인트를 무작위로 획득합니다. (일일 제한 엄격 적용)

## 3. 기술 스택 (Tech Stack)

| 구분 | 도입 기술 | 비고 |
| :--- | :--- | :--- |
| **Frontend Framework** | **React Native (Expo SDK 54)** | 크로스플랫폼 앱 개발 환경 구성 |
| **UI & Styling** | **Vanilla CSS + Expo LinearGradient** | 토스(Toss) 스타일의 프리미엄 일렉트릭 블루/사이안 테마 구현 |
| **Authentication** | **Firebase Auth** | 이메일/비밀번호 계정 인증 |
| **Database** | **Google Cloud Firestore** | 실시간 개인 지표, 설정값, 스토리 히스토리 영속화 |
| **Sensor API** | **Expo Sensors (Pedometer & Accelerometer)** | 실시간 보걸음 측정 및 기기 흔들기 꼼수 감지 |
| **AI Integration** | **OpenRouter API (LLM)** | AI 반성문 작문 및 분기형 웹소설 챕터 동적 생성 |

## 4. 데이터베이스 스키마 구조 (Firestore)

모든 사용자 정보는 로그인한 유저 고유의 `userId (auth.uid)`를 기준 문서(Document)로 관리합니다.

### 📁 `settings/{userId}` (설정 데이터)
```json
{
  "targetSteps": 10000,
  "nickname": "날렵한토끼312",
  "duoName": "홍길동",
  "duoPhone": "010-1234-5678",
  "warningContacts": [
    { "name": "엄마", "phone": "010-9999-8888" },
    { "name": "부장님", "phone": "010-7777-6666" }
  ]
}
```

### 📁 `userStats/{userId}` (운동 통계 & 인벤토리)
```json
{
  "todaySteps": 4200,
  "accumulatedSteps": 15600,
  "points": 230,
  "dailyGoalAchievements": 3,
  "todayGoalAchieved": false,
  "storyTheme": "romance_idol",
  "lastResetDate": "Sun Jul 19 2026",
  "lastRandomBoxDate": "Sun Jul 19 2026",
  "shieldActive": true,

```

## 5. 데모/시연용 특별 기능 (Demo Panel)

발표 및 심사위원 시연 시 실제 10,000보를 직접 걷기 힘든 환경을 배려해 **홈 화면 최하단에 노란색 배경의 `시연용 컨트롤 (Demo Controls)` 패널을 무조건적으로 활성화**했습니다.

1. **`+1,000보` / `-1,000보` 버튼**:
   * 즉각적으로 걸음 수를 조작하여 원형 링 게이지가 차오르고 달성률(%)이 갱신되는 역동적인 애니메이션 효과를 보여줄 수 있습니다.
   * 음수(-) 조작 또한 정상 지원하여 0 이하로 내려가지 않는 예외 안전을 보장합니다.
2. **`🏆 만보 달성일수 +1일` 버튼**:
   * 달성 일수가 늘어남에 따라 캐릭터의 경험치(XP) 바가 누적되고, 임계점을 돌파해 **Lv.1(돼지) → Lv.2(인간) → Lv.3(운동왕) → Lv.4(존예)** 캐릭터가 극적으로 진화(레벨업)하는 화면을 즉각 시연할 수 있습니다.
3. **오류 무해화**:
   * Firebase 연결이 불안정한 환경에서도 `둘러보기 (게스트 모드)`로 완벽히 동일한 기능을 시연할 수 있습니다.
   * 비정상 로그인 시도가 일어나도 붉은 에러 창을 띄우지 않고 깔끔한 노란색 주의 경고문으로 자연스럽게 대처합니다.
